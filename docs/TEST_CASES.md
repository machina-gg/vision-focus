# E2Eテストケース

## 1. テスト概要

| 項目              | 内容                                          |
| ----------------- | --------------------------------------------- |
| 対象ブラウザ      | Chromium（Desktop、headless: false 必須）     |
| テストツール      | Playwright                                    |
| 実行コマンド      | `npm run test:e2e`                            |
| 拡張機能ロード    | `--load-extension` オプションで dist をロード |
| 対象ページ        | newtab.html, options.html, popup.html         |

### Chrome拡張機能テストの制約

| 項目                 | 状況                                             |
| -------------------- | ------------------------------------------------ |
| ブラウザ             | Chromiumのみ（Firefox/WebKit不可）               |
| ヘッドレスモード     | 使用不可（`headless: false` 必須）               |
| Service Worker       | 直接テスト不可（ロジックは単体テストでカバー）   |
| `chrome.*` API       | 一部モックが必要                                 |
| 拡張機能ID           | ビルドごとに変わる可能性あり                     |

## 3. テストシナリオ詳細

### FLOW-001: ブロックリストにサイトを追加し、アクセスがブロックされる

**前提条件**

- 拡張機能がインストールされている
- ブロックリストが空

**手順**

1. オプション画面のブロックリストタブを開く
2. 「サイトを追加」ボタンをクリック
3. `twitter.com` を入力
4. 「追加」ボタンをクリック
5. 新規タブで `https://twitter.com` にアクセス

**期待結果**

- ブロックページが表示される
- 「twitter.com をブロック中」と表示される
- 目標テキストが表示される

---

### FLOW-002: 目標テキストを設定し、ダッシュボードに表示される

**前提条件**

- 拡張機能がインストールされている

**手順**

1. オプション画面のスタイル設定タブを開く
2. 目標テキスト欄に「ライバルを追い抜く」と入力
3. 変更が自動保存される
4. 新規タブを開く

**期待結果**

- ダッシュボードに「ライバルを追い抜く」と表示される

---

### FLOW-003: スケジュールを作成し、指定時間帯でブロックが有効になる

**前提条件**

- 拡張機能がインストールされている
- ブロックリストに `twitter.com` が登録されている

**手順**

1. オプション画面のスケジュールタブを開く
2. 「スケジュールを追加」ボタンをクリック
3. 名前: 「仕事時間」、開始: 09:00、終了: 18:00、曜日: 月〜金を選択
4. 「保存」をクリック
5. 現在時刻がスケジュール内の場合、`https://twitter.com` にアクセス

**期待結果**

- スケジュール時間内: ブロックページが表示される
- スケジュール時間外: 通常通りアクセスできる

---

### FLOW-004: ロックダウンモードを有効化し、全サイトがブロックされる

**前提条件**

- 拡張機能がインストールされている

**手順**

1. ポップアップを開く
2. 「ロックダウンモード」ボタンをクリック
3. 確認ダイアログで「有効化」をクリック
4. ブロックリスト外のサイト（例: `github.com`）にアクセス

**期待結果**

- ブロックページが表示される
- 解除チャレンジボタンが無効化されている

---

### BLOCK-006: 解除チャレンジ成功で一時的にサイトにアクセスできる

**前提条件**

- ブロックリストに `twitter.com` が登録されている
- ロックダウンモードが無効

**手順**

1. `https://twitter.com` にアクセス
2. ブロックページで「解除チャレンジ」をクリック
3. モーダルで指定されたテキストを入力
4. 「解除する」をクリック

**期待結果**

- `twitter.com` にアクセスできる
- 一定時間後（5分）に再びブロックされる

---

### OPT-ST-010: 新規スタイルを作成できる

**前提条件**

- 拡張機能がインストールされている
- 無料版の状態

**手順**

1. オプション画面のスタイル設定タブを開く
2. 「新規スタイル作成」ボタンをクリック
3. スタイル名に「仕事モード」と入力
4. 「作成」をクリック

**期待結果**

- スタイル一覧に「仕事モード」が追加される
- 作成したスタイルが選択状態になる

---

### OPT-SC-002: 週間カレンダーが表示される

**前提条件**

- スケジュールが1件以上登録されている

**手順**

1. オプション画面のスケジュールタブを開く

**期待結果**

- 週間カレンダーが表示される
- 縦軸に時間（0〜24時）、横軸に曜日（日〜土）が表示される
- 登録済みスケジュールが色付きブロックで表示される
- スタイル連携している場合、ブロック内にスタイル名が表示される

## 4. 優先度の定義

| 優先度 | 説明                                       | 目安件数 |
| ------ | ------------------------------------------ | -------- |
| P0     | 必須：クリティカルパス、リリースブロッカー | 約25件   |
| P1     | 重要：主要機能、エラーハンドリング         | 約35件   |
| P2     | 推奨：エッジケース、UX向上                 | 約15件   |

## 5. ステータスの定義

| ステータス | 説明                    |
| ---------- | ----------------------- |
| -          | 未実装                  |
| 実装済     | E2Eテストコード作成済み |
| Pass       | テスト成功              |
| Fail       | テスト失敗（要修正）    |

## 6. テスト実行環境

### Playwright設定（例）

```typescript
// playwright.config.ts
import { defineConfig } from '@playwright/test';
import path from 'path';

export default defineConfig({
  testDir: './e2e',
  timeout: 30000,
  use: {
    browserName: 'chromium',
    headless: false, // 拡張機能テストには必須
    launchOptions: {
      args: [
        `--disable-extensions-except=${path.join(__dirname, 'dist')}`,
        `--load-extension=${path.join(__dirname, 'dist')}`,
      ],
    },
  },
});
```

### 拡張機能ページへのアクセス

```typescript
// 拡張機能IDの取得
const extensionId = await getExtensionId(context);

// 各ページへのアクセス
await page.goto(`chrome-extension://${extensionId}/newtab.html`);
await page.goto(`chrome-extension://${extensionId}/options.html`);
```

## 7. 注意事項

- Service Workerのロジックは単体テスト（Vitest）でカバー
- `chrome.storage` のモック戦略を事前に決定
- CIでの実行は `xvfb` などの仮想ディスプレイが必要
- 拡張機能IDはビルドごとに変わる可能性があるため、動的取得が必要
