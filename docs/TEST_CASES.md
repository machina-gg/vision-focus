# E2Eテストケース

## 1. テスト概要

| 項目         | 内容                          |
| ------------ | ----------------------------- |
| 対象ブラウザ | Chrome（Desktop）             |
| テストツール | Playwright                    |
| 実行コマンド | `pnpm test:e2e`               |
| テスト対象   | Chrome拡張機能（Manifest V3） |

### テスト環境の注意点

Chrome拡張機能のE2Eテストには以下の特殊な設定が必要：

- Playwright の `chromium` を使用し、拡張機能をロード
- `--disable-extensions-except` と `--load-extension` フラグを使用
- Service Worker のテストには `chrome.runtime` API のモックが必要な場合あり

## 3. テストシナリオ詳細

### POP-001: ポップアップが正常に表示される

**前提条件**

- 拡張機能がインストールされている

**手順**

1. ツールバーのVisionFocusアイコンをクリック

**期待結果**

- ポップアップウィンドウが表示される
- 幅360px程度のウィンドウが開く

---

### BLK-001: ブロック対象サイトにアクセスするとブロックページが表示される

**前提条件**

- ブロックリストに「twitter.com」が登録されている

**手順**

1. ブラウザで「https://twitter.com」にアクセス

**期待結果**

- ブロックページが表示される
- 「BLOCKED」の文字が表示される
- 目標テキストが表示される

---

### BLK-008: チャレンジテキストを正しく入力すると解除される

**前提条件**

- ブロックページが表示されている
- ロックダウンモードがOFF

**手順**

1. 「解除チャレンジ」ボタンをクリック
2. モーダルに表示されたテキスト（例: "I choose to focus on my goals"）を入力
3. 「解除する」ボタンをクリック

**期待結果**

- モーダルが閉じる
- 元のサイト（twitter.com）が表示される
- 5分間はブロックされない

---

### BLOCK-001: ブロックリストに追加したサイトがブロックされる

**前提条件**

- ブロックリストが空

**手順**

1. オプション画面を開く
2. ブロックリストタブを選択
3. 「+ サイトを追加」をクリック
4. 「example.com」を入力して保存
5. 新しいタブで「https://example.com」にアクセス

**期待結果**

- ブロックページが表示される

---

### OPT-B03: ドメインを入力してブロックリストに追加できる

**前提条件**

- オプション画面が開いている
- ブロックリストタブが選択されている

**手順**

1. 「+ サイトを追加」ボタンをクリック
2. 入力フィールドに「reddit.com」を入力
3. 追加/保存ボタンをクリック

**期待結果**

- ブロックリストに「reddit.com」が追加される
- リストに新しい項目が表示される

---

## 4. 優先度の定義

| 優先度 | 説明                                                 | MVP対象 |
| ------ | ---------------------------------------------------- | ------- |
| P0     | 必須：クリティカルパス、コア機能、リリースブロッカー | ✅      |
| P1     | 重要：主要機能、エラーハンドリング、UX               | ✅      |
| P2     | 推奨：エッジケース、有料版機能、細かいUX             | -       |

## 5. ステータスの定義

| ステータス | 説明                    |
| ---------- | ----------------------- |
| -          | 未実装                  |
| 実装済     | E2Eテストコード作成済み |
| Pass       | テスト成功              |
| Fail       | テスト失敗（要修正）    |
| Skip       | スキップ（理由を記載）  |

## 6. テスト実装の優先順位

### Phase 1: コア機能（MVP必須）

1. BLOCK-001: サイトブロック基本動作
2. BLK-001, BLK-002, BLK-003: ブロックページ表示
3. POP-001, POP-003, POP-004: ポップアップ基本表示
4. NEW-001, NEW-002, NEW-003: ダッシュボード基本表示
5. OPT-B01, OPT-B03, OPT-B06: ブロックリスト管理

### Phase 2: 主要機能

1. BLK-007, BLK-008: 解除チャレンジ
2. POP-007, POP-008, POP-009: ロックダウンモード
3. TIME-001, TIME-003: 滞在時間計測
4. DATA-001, DATA-002: データ永続化

### Phase 3: 追加機能

1. OPT-S01〜S05: スケジュール機能
2. OPT-A01〜A04: 分析機能
3. I18N-001〜003: 多言語対応
