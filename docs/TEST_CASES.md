# E2Eテストケース

## 1. テスト概要

| 項目         | 内容                          |
| ------------ | ----------------------------- |
| 対象ブラウザ | Chrome（Desktop）             |
| テストツール | Playwright                    |
| 実行コマンド | `pnpm test:e2e`               |
| テスト対象   | Chrome拡張機能（Manifest V3） |

### テスト環境の注意点

Chrome拡張機能のE2Eテストには以下の特殊な設定が必要：

- Playwright の `chromium` を使用し、拡張機能をロード
- `--disable-extensions-except` と `--load-extension` フラグを使用
- Service Worker のテストには `chrome.runtime` API のモックが必要な場合あり

## 3. テストシナリオ詳細

### POP-001: ポップアップが正常に表示される

**前提条件**

- 拡張機能がインストールされている

**手順**

1. ツールバーのVisionFocusアイコンをクリック

**期待結果**

- ポップアップウィンドウが表示される
- Header（ロゴ・設定・Pauseトグル・言語選択）が表示される
- QuickBlockButton が表示される
- GoalCard が表示される
- 今日のサマリー（ブロック回数・トップブロックサイト）が表示される

---

### NEW-001: 新規タブを開くとダッシュボードが表示される

**前提条件**

- プリセットが1件以上設定されている

**手順**

1. 新規タブを開く

**期待結果**

- 背景画像が表示される
- 目標テキストが中央に表示される
- ミニ統計カードが表示される
- 設定アイコンが右下に表示される

---

### NEW-008: プリセット未設定時、シンプルなブロックページUIが表示

**前提条件**

- プリセットが1件も設定されていない

**手順**

1. 新規タブを開く

**期待結果**

- シンプルな背景色（#1a1a2e）が表示される
- VisionFocus ロゴと「目標を思い出そう」メッセージが表示される
- ミニ統計カードが表示される
- ブロックサイトリストが表示される
- 「初めてのスタイルを作成」CTA が表示される

---

### BLOCK-001: ブロックリストに追加したサイトが newtab.html にリダイレクト

**前提条件**

- ブロックリストが空

**手順**

1. オプション画面を開く
2. ブロックリストタブを選択
3. 入力フィールドに「example.com」を入力
4. Enter キーまたは追加ボタンをクリック
5. 新しいタブで「https://example.com」にアクセス

**期待結果**

- newtab.html にリダイレクトされる
- ブロック情報（domain, count）が表示される
- 「サイトがブロックされました」メッセージが表示される

---

### OPT-B02: ドメインを入力してブロックリストに追加できる

**前提条件**

- オプション画面が開いている
- ブロックリストタブが選択されている

**手順**

1. 入力フィールドに「reddit.com」を入力
2. Enter キーまたは追加ボタンをクリック

**期待結果**

- ブロックリストに「reddit.com」が追加される
- リストに新しい項目が表示される
- ドメインリストアイテムにブロックスタイル（Always Blocked）が表示される

---

### TL-003: Time Limit 超過時に newtab.html へリダイレクト

**前提条件**

- ブロックリストに「youtube.com」が登録されている
- Daily Time Limit が 10秒 に設定されている
- 10秒以上 YouTube に滞在している

**手順**

1. YouTube にアクセスする
2. 10秒以上待つ

**期待結果**

- newtab.html へリダイレクトされる
- URL パラメータに `?reason=time_limit_exceeded` が含まれる
- 「時間制限に達しました」メッセージが表示される

### OPT-S10: 設定した時間帯に該当プリセットが自動適用される

**前提条件**

- プリセット「仕事モード」が作成されている
